ISSUE #4: AX.25 DECODER CRC ERROR
════════════════════════════════════════════════════════════════════

Error Message:
  "Size inputs must be integers"
  
Location:
  calculateCRC16 (line 148)
  Called from ax25_decode (line 75)

Cause:
  numBytes = length(bits) / 8
  
  If bit array length is not exactly divisible by 8 (e.g., 1201 bits),
  this creates a non-integer value (150.125), which can't be used
  as an array size.


THE FIX
════════════════════════════════════════════════════════════════════

Changed in BOTH ax25_encode.m and ax25_decode.m:

OLD CODE (WRONG):
  numBytes = length(bits) / 8;
  bytes = zeros(1, numBytes);  % ERROR if numBytes not integer!

NEW CODE (CORRECT):
  numBytes = floor(length(bits) / 8);  % Force integer
  bytes = zeros(1, numBytes);
  
Also added bounds checking:
  endIdx = startIdx + 7;
  if endIdx <= length(bits)
      byte_bits = bits(startIdx:endIdx);
      bytes(i) = bits2byte(byte_bits);
  end
