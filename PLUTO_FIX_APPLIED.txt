PROBLEM IDENTIFIED
════════════════════════════════════════════════════════════════════

The diagnostic showed that:
  ✓ Pluto IS connected and working
  ✓ MATLAB CAN detect it (found RadioID: usb:0)
  ✓ All toolboxes and functions are available

BUT the GUI wasn't showing it because:
  ✗ gui_app.m line 141 was capturing the WRONG return value
  
  OLD CODE (WRONG):
    app.availableRadios = pluto_config();
    
  This captured txSDR (first output), not availableRadios (third output)


THE FIX
════════════════════════════════════════════════════════════════════

FIXED CODE:
  [~, ~, app.availableRadios] = pluto_config();
  
This now correctly captures the third output (availableRadios).

SINGLE PLUTO USAGE
════════════════════════════════════════════════════════════════════

Since you have 1 Pluto, you can:

Option A: Loopback Test (TX → RX on same radio)
  • Connect TX to RX with attenuator
  • Use same radio for both TX and RX
  • Good for testing signal processing

Option B: One-Way Communication
  • Configure as TX only
  • Send to another receiver (RTL-SDR, etc.)
  
Option C: Get Second Pluto
  • For full duplex communication
  • Each station has own radio
